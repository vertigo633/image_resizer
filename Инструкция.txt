Требуется реализовать ресайзер картинок. Клиент должен иметь возможность отправлять картинку в формате jpg POST-запросом в сервис по следующему url:
POST /api/v1/picture-resizer/<id>/upload
При запросе следующих ресурсов
GET /api/v1/picture-resizer/<id>/get/size1
GET /api/v1/picture-resizer/<id>/get/size2
GET /api/v1/picture-resizer/<id>/get/size3
GET /api/v1/picture-resizer/<id>/get/size4
должны выдаваться ресайзы загруженного изображения, где size1, size2, size3, size4 - названия различных размеров нарезки для картинок (размеры - любые на выбор). При запросе такого ресурса в первый раз контроллер должен генерировать ресайз изображения, в последующие разы - отдавать уже сгенерированное изображение. Идентификатор изображения назначается самим клиентом, т.е. первый POST-запрос создает запись с указанным в запросе id.
Выполнить задание необходимо с использованием следующих технологий:
Spring Framework
H2 или PostgreSQL
Библиотека ресайза - любая на выбор




Инструкция: 

До запуска:
1. Для запуска программы необходим Tomcat 7.*, jdk 1.7, БД PostegreSQL. Конфигурация БД задается в файле web/WEB-INF/jdbc.properties.
2. При обработке BufferedImage временный файл сохраняется в папке temp внутри контейнера Tomcat. Если Tomcat находится в Program Files, то может возникнуть ошибка доступа. Необходимо перенести каталог с Tomcat в другое место.  

Комментарий: 
Пытался решить проблему - менял настройки JVM, указывая в ней другую папку для временных файлов, менял Classpath - бесполезно. Самым простым решением оказалось перенести каталог с Tomcat из Program Files и перезапустить сервер. 

После запуска: 

1. После запуска программы и после того как Hibernate создаст таблицы в БД необходимо выполнить запрос INSERT: 

INSERT INTO size(
            name, height, width)
    VALUES ('size1', 800, 600), ('size2', 600, 400), ('size3', 400, 200), ('size4', 200, 100); 
	
Комментарий: 
Для первоначального создания таблицы размеров (сущность Size) самым простым решением оказалось сделать это вручную, т.к размеры не нужно задавать каждый раз при запуске программы.  

2. Для загрузки изображений на сервер можно воспользоваться стартовой страницей программы (index.jsp)
